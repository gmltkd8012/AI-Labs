## LLM (Large Language Model)

### 정의
대규모의 텍스트 데이터를 학습하여 주어진 질문이나 문장에 이어질 내용을 생성하는 인공지능 모델

### 작동 방식
```
입력 텍스트 → [학습된 패턴 분석] → 다음 단어/문장 생성 → 출력
```

### 특징
- 방대한 데이터셋으로 학습됨
- 자연스러운 텍스트 생성 능력
- 다양한 작업 수행 가능 (번역, 요약, 질답 등)

### 한계 ⚠️
- **지식의 고정성**: 학습된 데이터 내에서만 답변 가능
- **최신 정보 부재**: 학습 이후의 새로운 정보 반영 불가
- **할루시네이션(Hallucination)**: 잘못된 정보를 사실처럼 생성할 수 있음

### 예시
```
사용자: "한국의 현재 대통령은?"
LLM: (학습 시점의 정보로만 답변)
```

## RAG (Retrieval-Augmented Generation)

### 정의
답변 제공 시 실시간으로 문서나 데이터베이스에서 관련 정보를 검색하여, LLM이 그 문서 기반으로 정확한 답변을 생성하도록 하는 기술

### 작동 방식
```
사용자 질문 → [정보 검색] → 관련 문서 검색 → LLM이 검색된 자료로 답변 생성 → 출력
```

### 구성 요소
1. **Retriever** (검색기): 관련 문서/정보 검색
2. **Generator** (생성기): LLM이 검색 결과를 기반으로 답변 생성

### 특징
- LLM의 한계 극복
- 답변의 신뢰성과 정확도 향상
- 최신 정보 반영 가능
- 출처 제시 가능

### 한계 ⚠️
- **실시간 변동 데이터**: 실시간으로 빠르게 변하는 정보 처리 어려움
  - 날씨 정보
  - 주가/암호화폐 가격
  - 실시간 뉴스
  - 거래소 환율

### 예시
```
사용자: "오늘 서울의 날씨는?"
RAG 시스템:
  1. 날씨 정보 데이터베이스 검색
  2. 실시간 날씨 데이터 추출
  3. LLM이 그 정보 기반으로 답변 생성
```

## AI Agent

### 정의
LLM 및 RAG를 포함한 다양한 기술을 활용하여 사용자 질문을 처리하고 실제 행동을 수행할 수 있는 지능형 시스템. 다양한 플러그인과 외부 API를 활용하여 자율적으로 작동

### 작동 방식 (순환 구조)
```
┌─────────────────────────────────────────┐
│                                         │
│  1️⃣ 사용자 질문 파악                    │
│  ↓                                      │
│  2️⃣ 계획 수립 (어떤 도구를 사용할지)  │
│  ↓                                      │
│  3️⃣ 도구 활용 (API, 함수 호출)         │
│  ↓                                      │
│  4️⃣ 결과 분석 및 반복 수행             │
│  ↓                                      │
│  최종 답변 또는 행동 수행                │
│                                         │
└─────────────────────────────────────────┘
```

### 핵심 요소

#### 1. 인식 (Perception)
- 사용자의 질문/명령 이해
- 자연어 처리를 통한 의도 파악

#### 2. 계획 (Planning)
- 목표 달성을 위한 전략 수립
- 어떤 도구/API를 사용할지 결정
- 실행 순서 결정

#### 3. 실행 (Execution)
- 외부 API 호출
- 도구/플러그인 활용
- 실제 작업 수행 (이메일 발송, 일정 추가 등)

#### 4. 반복 (Iteration)
- 결과 평가
- 필요시 계획 수정 후 재실행
- 최종 답변 전달

### 특징
- **자율성**: 사용자의 직접 개입 없이 자동으로 작동
- **도구 활용**: 다양한 외부 API와 플러그인 통합
- **적응성**: 상황에 따라 전략 변경
- **복합 작업**: 여러 단계의 작업을 자동으로 처리

### 활용 예시

#### 예시 1: 스마트 어시스턴트
```
사용자: "내일 오후 3시에 서울에서 부산으로 가는 기차표 예약하고, 
        부산의 날씨를 확인한 후 날씨에 맞는 옷 추천해줘"

AI Agent의 동작:
1️⃣ 사용자 의도 파악
   → 기차 예약, 날씨 확인, 옷 추천

2️⃣ 계획 수립
   → (1) 기차 예약 API 호출
   → (2) 날씨 API 호출
   → (3) 옷 추천 로직 실행

3️⃣ 도구 활용
   → 기차 예약 시스템 접근
   → 날씨 데이터 베이스 검색
   → 추천 알고리즘 실행

4️⃣ 반복
   → 기차 예약 성공? → 날씨 확인 진행
   → 날씨 데이터 수신? → 옷 추천 생성

최종 결과:
✅ 기차표 예약 완료
✅ "내일 부산은 맑고 따뜻합니다. 반팔과 가벼운 바지를 추천합니다."
```

#### 예시 2: 마케팅 자동화
```
사용자: "이번 분기 마케팅 캠페인 계획 수립 및 실행"

AI Agent의 동작:
1️⃣ 질문 파악: 캠페인 계획과 실행

2️⃣ 계획 수립
   → 시장 분석 API 호출
   → 고객 데이터 분석
   → 캠페인 전략 생성
   → 타겟팅 설정

3️⃣ 도구 활용
   → 광고 플랫폼 API (Google Ads, Meta 등)
   → 이메일 마케팅 도구
   → CRM 시스템
   → 분석 대시보드

4️⃣ 반복
   → 캠페인 실행
   → 성과 모니터링
   → 필요시 최적화
```

## 세 개념의 관계도

### 진화 과정
```
┌─────────┐
│   LLM   │  (기본: 텍스트 생성)
└────┬────┘
     │
     │ 한계 극복
     ▼
┌─────────┐
│   RAG   │  (개선: 실시간 정보 활용)
└────┬────┘
     │
     │ 확장
     ▼
┌──────────────┐
│  AI Agent    │  (완전: 자율 행동 수행)
└──────────────┘
```

### 계층 관계
```
AI Agent (최상위)
├── LLM (핵심: 텍스트 이해 및 생성)
├── RAG (보조: 정보 검색 및 보강)
└── 외부 도구 & API (실행: 실제 작업 수행)
```

### 비교 표

| 구분 | LLM | RAG | AI Agent |
|------|-----|-----|----------|
| **역할** | 텍스트 생성 | 정보 검색 + 생성 | 자율 의사결정 + 실행 |
| **정보 출처** | 학습 데이터만 | 실시간 DB + 학습 | 다양한 소스 + 도구 |
| **최신성** | 낮음 | 높음 | 매우 높음 |
| **상황 대응** | 제한적 | 중간 | 우수 |
| **자동 실행** | 불가능 | 불가능 | 가능 |
| **복잡도** | 낮음 | 중간 | 높음 |
| **비용** | 낮음 | 중간 | 높음 |

## 참고: 용어 정리

- **Hallucination**: LLM이 학습하지 않은 내용을 마치 사실인 것처럼 생성하는 현상
- **Embedding**: 텍스트를 수치 벡터로 변환하여 의미를 표현하는 방법
- **Retriever**: 관련 문서나 정보를 검색하는 시스템
- **Generator**: 검색된 정보를 바탕으로 답변을 생성하는 LLM
- **API**: 외부 시스템과 통신하여 기능을 이용하는 인터페이스
